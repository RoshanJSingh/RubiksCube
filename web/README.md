# Rubik's Cube Solver — Web (C++ → WebAssembly)

An interactive 3D front-end for the C++ solver in this repo. The **C++ engine is
the brain**: it is compiled to WebAssembly with Emscripten and runs entirely in
the visitor's browser — no server, no backend, no cold starts. The front-end
(Three.js) only renders the cube and animates the moves the engine returns.

```
web/
├── public/            # the static site that gets deployed
│   ├── index.html
│   ├── style.css
│   ├── app.js         # Three.js cube + animation + WASM glue
│   └── rubiks.mjs / rubiks.wasm   # generated by build.sh (not committed)
├── src/
│   ├── cube_solver.hpp   # WebCube wrapper over the existing model + IDDFS solver
│   ├── bindings.cpp      # Emscripten/embind bindings (compiled to WASM)
│   ├── test_native.cpp   # native g++ test harness (correctness + timing)
│   └── shims/bits/stdc++.h  # <bits/stdc++.h> replacement for Emscripten/libc++
└── build.sh           # emcc build command
```

## How it works

- `WebCube` (in `src/cube_solver.hpp`) wraps `RubiksCubeBitboard` and the project's
  `IDDFSSolver`. It exposes `reset()`, `applyMoves("U R' F2 ...")`, `state()`,
  `solved()` and `solve(maxDepth)` to JavaScript.
- The browser generates a random scramble, sends it to the engine, and the engine
  returns an **optimal** (shortest) solution via iterative-deepening DFS.
- Solve times grow fast without a pattern database, so the UI caps scramble depth
  at 6 (depth 5 is instant; depth 6 takes a few seconds). For deeper scrambles the
  IDA\* + corner pattern-database path (a server build) is the way to go.

## Build locally (optional)

You only need this if you want to preview the WASM locally; CI builds it for you.

1. Install the [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html)
   so that `emcc` is on your `PATH`.
2. From the repo root:
   ```bash
   bash web/build.sh
   ```
   This produces `web/public/rubiks.mjs` and `web/public/rubiks.wasm`.
3. Serve the folder over HTTP (WASM won't load from `file://`):
   ```bash
   python -m http.server -d web/public 8080
   ```
   Then open <http://localhost:8080>.

### Verify the C++ side without Emscripten

```bash
g++ -std=c++14 -O2 -o web/src/test_native web/src/test_native.cpp
./web/src/test_native      # prints solve times and verifies each solution
```

## Deploy to GitHub Pages (free)

The workflow at `.github/workflows/deploy.yml` compiles the WASM and publishes
`web/public/` on every push to `main`.

1. Push this repo to GitHub (already at `RoshanJSingh/RubiksCube`).
2. In the repo: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
3. Push to `main` (or run the workflow manually from the **Actions** tab).
4. The site goes live at `https://roshanjsingh.github.io/RubiksCube/`.

No secrets or paid services required.
